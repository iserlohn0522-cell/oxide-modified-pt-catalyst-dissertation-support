# Privacy and Exclusions

The public package is organized around scientific functions rather than internal project history. It therefore omits source-image and sample identities, private data partitions, original microscopy, editable annotations, production masks and per-particle records, full cluster outputs, scheduler metadata, internal paths, failed calculations, working logs, dissertation files, and review material.

Custom weights are sanitized release assets. The Git repository blocks model binaries, archives, images, office/PDF files, and DFT coordinates outside the curated case directories. Text scanning rejects local paths, job identifiers, credentials, and unresolved review placeholders. Release assets are scanned separately before publication.

Selected DFT structures and aggregate Chapter 5 metrics are included because they support public inspection without exposing the underlying image archive or internal workflow history.
